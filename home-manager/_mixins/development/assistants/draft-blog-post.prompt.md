## Draft Blog Post

Write a blog post based on referenced content or topic.

### Gather (ask if not provided)

- **Topic**: What are we writing about?
- **Angle**: Hook or perspective
- **Length**: Short (~800w) | Standard (~1500w) | Long (~2500w)

Assume Linux-savvy audience unless specified.

### Post Types

| Type | Structure |
|------|-----------|
| Tutorial | Problem → Solution → Steps → Results |
| Opinion | Hook → Position → Evidence → Call to action |
| Announcement | News → Why it matters → What's next |
| Retrospective | Context → Journey → Lessons → Future |

### Process

1. Confirm topic, angle, length (or infer from context)
2. Draft applying voice calibration from agent definition
3. Present draft with word count
4. Iterate on feedback

### Example Brief

<example>
**Topic**: Switching from Ubuntu to NixOS
**Angle**: Confession/journey—why I made the leap after years of advocacy
**Length**: Standard (~1500w)
</example>
